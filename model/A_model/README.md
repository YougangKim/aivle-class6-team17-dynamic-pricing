# A Model

## 개요

A Model은 신선식품 동적 할인 시스템의 **가격 의사결정 모델**입니다.

기본수요를 예측한 뒤, 운영 가능한 할인 후보를 생성하고 시뮬레이션 결과를 바탕으로 최적의 할인 정책을 선택합니다.

A Model은 다음과 같이 구성됩니다.

- **기본수요 예측 모델 (Base Demand Forecasting Model)**
  - 정가 기준 기본수요(P10·P50·P90)를 예측합니다.

- **할인 후보 생성 모듈 (Discount Candidate Generation Module)**
  - 재고, 유통기한, 가격탄력성 및 운영 제약을 고려하여 할인 후보를 생성합니다.

- **최적 할인 정책 선택 모듈 (Discount Policy Selection Module)**
  - 정책 시뮬레이션 결과를 바탕으로 최적의 할인 정책을 선택합니다.

# 모델 흐름

```
판매 데이터
      │
      ▼
기본수요 예측 모델
      │
      ▼
할인 후보 생성 모듈
      │
      ▼
B 모델 정책 시뮬레이션
      │
      ▼
최적 할인 정책 선택 모듈
```

---

# 모델 입·출력

## 1. 기본수요 예측 모델

### 입력

- 점포·상품·시간 정보
- 달력 정보(요일, 공휴일, 계절 등)
- 가격 및 원가 정보
- 방문객 정보
- 재고 및 잔여 유통기한 정보
- 과거 판매 시계열 피처

### 출력

| 출력 | 설명 |
|------|------|
| store_id | 점포 ID |
| product_id | 상품 ID |
| date | 예측 날짜 |
| hour | 예측 시간 |
| base_demand_p10 | 저수요 상황 기본수요 |
| base_demand_p50 | 일반적인 기본수요 |
| base_demand_p90 | 고수요 상황 기본수요 |

---

## 2. 할인 후보 생성 모듈

### 입력

- 기본수요(P10·P50·P90)
- 현재 재고
- 잔여 유통기한
- 정가 및 원가
- 가격탄력성
- 할인 운영 제약조건
- 기존 할인 정책

### 출력

후보 할인 정책 테이블

| 출력 | 설명 |
|------|------|
| policy_id | 후보 정책 ID |
| store_id | 적용 점포 |
| product_id | 적용 상품 |
| date | 적용 날짜 |
| hour | 적용 시간 |
| dte_min | 최소 잔여 유통기한 |
| dte_max | 최대 잔여 유통기한 |
| discount_rate | 적용 할인율 |

---

## 3. 최적 할인 정책 선택 모듈

### 입력

- 할인 후보 정책
- B 모델 정책 시뮬레이션 결과

### 출력

최종 할인 정책

| 출력 | 설명 |
|------|------|
| selected_policy_id | 선택된 정책 |
| recommended_start_hour | 권장 할인 시작 시점 |
| recommended_discount_rate | 권장 할인율 |
| expected_sales_qty | 예상 판매량 |
| expected_revenue | 예상 매출 |
| expected_margin | 예상 마진 |
| expected_waste_qty | 예상 폐기량 |
| expected_profit | 예상 이익 |
| improvement_vs_existing | 기존 정책 대비 개선 효과 |
